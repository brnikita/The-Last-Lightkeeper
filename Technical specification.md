## 1. Концепция игры
**Название:** «Последний Смотритель» (The Last Lightkeeper).
**Жанр:** сюжетная одиночная игра от третьего лица, исследование + лёгкие головоломки. НЕ хоррор. Настроение: меланхоличное, тёплое, атмосферное — как Firewatch, RiME, Journey.
**Сеттинг:** небольшой заброшенный остров с маяком после того, как жители уехали. Главный герой — Эли, молодой смотритель, вернувшийся, чтобы зажечь маяк в последний раз и разобраться с воспоминаниями о деде-смотрителе.
**Сюжет:**
- **Завязка:** Эли прибывает на остров на лодке; маяк потушен. Чтобы его зажечь, нужно найти 4 фрагмента линзы Френеля, спрятанные по острову.
- **Развитие:** исследование пристани, деревни, леса и бухты; через найденные предметы, записки и голос деда (старые радиозаписи) постепенно раскрывается история их отношений.
- **Кульминация:** подъём на вершину маяка в надвигающийся шторм, сборка линзы из фрагментов.
- **Развязка:** маяк зажигается, прорезая шторм; вдалеке появляется корабль; тёплое эмоциональное прощание Эли с островом.
**Персонажи (для озвучки ElevenLabs, русская речь):**
- **Эли** — внутренний монолог, тёплый молодой мужской голос. Пример реплики: «Дедушкин фонарь… он всё ещё здесь. Кажется, будто он просто вышел на минуту.»
- **Дед** — через старые аудиозаписи/радио, пожилой, хриплый, добрый. Пример: «Свет — это не про то, чтобы видеть, малыш. Это про то, чтобы кто-то знал: его ждут дома.»
- **Голос с радио** — смотрительница соседнего острова, женский, спокойный. Пример: «Эли? Ты всё-таки вернулся. Зажги его. Ради него.»

## 3. Геймплейные механики
- **Передвижение от третьего лица:** WASD + мышь, **kinematic character controller на Rapier** (`world.createCharacterController(0.01)`, autostep для ступенек, snap-to-ground, slope limit). Позицию персонажа двигает контроллер, не анимация.
- **Камера:** third-person follow с **collision через raycast/boxcast** — луч от игрока к желаемой позиции камеры; при попадании в стену камера ставится в точку попадания (используй boxcast по размеру near-plane, чтобы не клипнуть в упор к стене).
- **Взаимодействие:** raycast от игрока/из центра экрана; подсветка интерактивных объектов (outline/эмиссия); клавиша E.
- **Нарративная/диалоговая система:** JSON-сценарий (`dialogues.json`), текст в нижней части экрана + синхронная аудиоозвучка (Howler), нелинейный показ записок.
- **Инвентарь:** 4 фрагмента линзы + ключевые предметы; простая UI-панель.
- **Сохранения:** localStorage (позиция игрока, собранные предметы, прогресс глав, открытые двери, прослушанные записи).
- **Катсцены:** скриптованные перемещения камеры + затемнение + статичные иллюстрации (GPT Image 2).

## 4. Графика (пайплайн)
- Рендерер: `renderer.outputColorSpace = THREE.SRGBColorSpace`. Тонмаппинг — **через postprocessing** (`ToneMappingEffect` в режиме `ACES_FILMIC`, последним эффектом в цепочке), при этом `renderer.toneMapping = THREE.NoToneMapping` (иначе двойной тонмаппинг).
- Освещение: **HDRI (Poly Haven) как environment** + один `DirectionalLight` (солнце) с тенями (`PCFSoftShadowMap`, `shadow.mapSize = 2048`, аккуратно настроенный shadow-camera frustum под размер уровня) + `HemisphereLight`/ambient для подсветки теней.
- PBR-материалы: `MeshStandardMaterial` с картами albedo/normal/roughness/AO.
- Постпроцессинг (pmndrs/postprocessing `EffectComposer`): `RenderPass` → SSAO (умеренный) → Bloom (`luminanceThreshold ≈ 0.9`, низкая `intensity`) → ToneMapping(ACES_FILMIC) → опц. Vignette / лёгкий DepthOfField. **ToneMapping всегда последний.**
- Туман: **цветной градиентный туман в стиле Firewatch** (FogExp2 или кастомный шейдер, где цвет тумана зависит от расстояния/высоты) — даёт атмосферу и скрывает дальнюю геометрию (бонус к производительности).
- Партиклы: пыль в лучах света, листья на ветру, брызги у воды, искры/световой конус маяка.
- Скайбокс: HDRI Poly Haven (приоритет) либо сгенерированная GPT Image 2 equirectangular-панорама на inverted sphere (проверять бесшовность).

## 5. Звук (ElevenLabs + бесплатные источники)
- **Озвучка диалогов:** ElevenLabs Text-to-Speech API через SDK `@elevenlabs/elevenlabs-js`, модель `eleven_multilingual_v2` (русская речь), отдельный voice ID на каждого персонажа. Заранее сгенерировать все реплики в mp3, сложить в `/public/assets/audio/vo`, проигрывать через Howler (НЕ дёргать API в рантайме).
- **SFX:** ElevenLabs Sound Effects API (`POST https://api.elevenlabs.io/v1/sound-generation`, модель `eleven_text_to_sound_v2`). По документации: `duration` «must be at least 0.5 and at most 30» секунд; `loop` доступен только для `eleven_text_to_sound_v2`; стоимость — 40 кредитов за секунду при заданной длительности. Используй `loop=true` для эмбиента (ветер, волны), обычные клипы для шагов/дверей/скрипа маяка.
- **Музыка:** ElevenLabs Music API (Music v2) для фоновых треков, либо бесплатно Kevin MacLeod / Pixabay Music.
- **Реализация:** Howler.js с позиционным 3D-звуком (Web Audio panner) — источники привязаны к объектам (маяк, прибой, радио), громкость и панорама зависят от позиции игрока.
- **Лицензия:** на free-tier ElevenLabs **нет коммерческой лицензии и требуется атрибуция ElevenLabs**; для коммерческого релиза нужен минимум план Starter (промо ~$5/мес, стандартно $6/мес), который добавляет Commercial License, Instant Voice Cloning и 30 000 кредитов/мес (~30 мин TTS).

## 7. КРИТИЧЕСКИ ВАЖНО — чек-лист «детских болезней» AI-игр и как их исключить
Реализуй ВСЕ меры; в конце каждого этапа прогоняй релевантную часть чек-листа.

**Физика и коллизии:**
- [ ] Игрок НЕ проваливается сквозь пол: пол и вся статика имеют корректные коллайдеры (trimesh для сложной геометрии окружения через Rapier; капсульный кинематический коллайдер игрока).
- [ ] Достаточный фиксированный физический шаг, чтобы не «протуннелировать» сквозь тонкие стены на высокой скорости.
- [ ] Snap-to-ground включён: персонаж не «парит» над полом и не дрожит на склонах.
- [ ] Нет застревания в геометрии: autostep для ступенек, slope limit, зазор offset 0.01.
- [ ] Триггеры (зоны взаимодействия, переходы уровней) — отдельные sensor-коллайдеры Rapier, реально срабатывают (логировать в dev).

**Камера:**
- [ ] Камера не проходит сквозь стены: raycast/boxcast от игрока к камере, подтягивание при попадании.
- [ ] Камера не клипует near-plane сквозь объекты в упор (boxcast по размеру near-plane).

**Анимации:**
- [ ] Нет T-поз: для каждого состояния (idle/walk/run/interact) есть анимация; кроссфейд через `AnimationMixer`; фолбэк на idle, если клип не загрузился.
- [ ] Mixamo-анимации — **in-place (без root motion)**, т.к. позицию двигает контроллер; иначе будет «скольжение»/дрейф персонажа.

**Масштаб и размещение:**
- [ ] Единый масштаб: 1 unit = 1 метр; проверять импортированные модели (Mixamo часто экспортирует в масштабе 0.01).
- [ ] Персонаж стоит ногами на полу (выравнивание pivot), а не врастает/не парит.

**Производительность:**
- [ ] Бюджеты: цель **< 100 draw calls** (InstancedMesh для деревьев/камней/травы), разумный полигонбюджет, текстуры через KTX2/Basis, геометрия через Draco.
- [ ] LOD для дальних объектов; frustum culling (по умолчанию); туман скрывает дальнюю геометрию.
- [ ] Цель 60 FPS на десктопе, ≥30 FPS на среднем ноутбуке; контроль через stats.js и `renderer.info`.

**Память (утечки):**
- [ ] При выгрузке уровня — `dispose()` для geometry, material, texture; очистка AnimationMixer; удаление объектов из сцены.
- [ ] Object pooling для часто создаваемых объектов (партиклы, подбираемые предметы).

**Рендеринг-артефакты:**
- [ ] Нет z-fighting: near ≥ 0.1, без копланарных поверхностей; при больших дистанциях — `logarithmicDepthBuffer`.
- [ ] Прозрачность: `transparent=true`, `depthWrite=false` для альфы, корректный порядок отрисовки.

**Надёжность загрузки:**
- [ ] Фолбэки: если ассет не загрузился — заглушка (примитив/дефолтная текстура) + лог, игра не падает.
- [ ] Прелоадер с прогрессом (`LoadingManager`); старт игры только после загрузки критичных ассетов.

**Мобильные:**
- [ ] Детект устройства; на мобильных — пониженный `pixelRatio`, отключение тяжёлого постпроцессинга, упрощённые тени; виртуальный джойстик.

## Recommendations
1. **Идти строго по этапам 0→8**, не перепрыгивая; приёмка каждого этапа обязательна — это главный способ не накопить «детские болезни».
2. **Приоритет CC0-ассетам** (Poly Haven, KayKit, Quaternius, Kenney): бесплатно и без юридических рисков. CC-BY со Sketchfab использовать только при готовности указывать атрибуцию; CC0-фильтр там сильно сужает выбор (~2 000 моделей).
3. **ElevenLabs:** прототип на free-tier ($0), но перед любой публикацией перейти на Starter (~$5–6/мес) ради коммерческой лицензии — на free-tier её нет и требуется атрибуция.
4. **Целиться в стилизованный реализм, а не фотореализм.** Сильное HDRI-освещение + ACES + bloom + цветной туман на простых CC0-моделях дадут «дорогую» картинку дешевле и надёжнее, чем попытки фотореализма.
5. **Пороги, меняющие план:** если FPS < 30 — резать постпроцессинг, тени и полигоны (instancing, LOD), а не добавлять эффекты; если draw calls > 100 — объединять геометрию/инстансить; если Claude Code «застрял» на шейдере/композиции — закладывать дополнительные итерации (это известная слабость ИИ в визуально-пространственных задачах).

## Caveats
- **WebGPU пока экспериментален** (официальная формулировка Three.js — «still in an experimental state»); в продакшене остаёмся на WebGLRenderer. Соответственно postprocessing (pmndrs) совместим — он только для WebGL.
- Claude Code хуже работает с визуально-пространственными задачами (шейдеры, композиция, «ощущение» сцены), чем с логикой — закладывайте больше итераций и присылайте ему скриншоты для оценки.
- Версии pre-1.0 пакетов (Rapier 0.19.x, three-mesh-bvh 0.9.x) могут ломать API между минорами — пинить точные версии.
- Сложные модели со Sketchfab часто требуют оптимизации (полигоны, текстуры, KTX2/Draco) перед загрузкой в браузер.
- Часть деталей (точные текущие версии npm-пакетов, статус WebGPU) проверены на 12 июня 2026 и могут измениться — Claude Code должен сверить актуальные версии при `npm install`.
- Цены ElevenLabs в разных источниках расходятся ($5 промо против $6 стандарт за Starter; для Creator встречаются и $11, и $22) — уточнить на elevenlabs.io/pricing на момент покупки.
